\## 1. Dasar Teori



\*\*Git\*\* merupakan sistem pengelolaan versi (\*Version Control System\*) yang bersifat terdistribusi dan banyak digunakan oleh pengembang perangkat lunak untuk mencatat serta memantau perubahan yang terjadi pada file selama proses pengembangan. Dengan menggunakan Git, setiap perubahan pada kode dapat terdokumentasi dengan baik sehingga memudahkan proses kolaborasi antar pengembang.  



Sementara itu, \*\*GitHub\*\* adalah layanan berbasis web yang menyediakan hosting repositori Git sehingga proyek dapat disimpan secara online, dikelola, serta dibagikan dengan lebih mudah.



\*\*Command Line Interface (CLI)\*\* merupakan antarmuka berbasis teks yang memungkinkan pengguna menjalankan berbagai perintah langsung melalui terminal atau command prompt. Dalam kegiatan praktikum ini, CLI dimanfaatkan untuk menjalankan perintah-perintah Git sehingga proses pengelolaan repositori dapat dilakukan secara lebih cepat dan efisien.

\## 2. Setup Repository via CLI



Berikut adalah tahapan yang dilakukan untuk melakukan inisialisasi dan pengaturan repositori dari komputer lokal ke GitHub menggunakan CLI.



\### Langkah 1: Membuat Repositori Baru di GitHub



!\[Langkah 1]
(./assets/repo.png)



Tahap pertama yang dilakukan adalah membuat repositori baru pada GitHub. Repositori tersebut berfungsi sebagai tempat penyimpanan proyek secara online sehingga seluruh file dan kode yang dibuat dapat disimpan, dikelola, serta diakses melalui internet.



\### Langkah 2: Panduan Perintah Git



!\[Langkah 2]
(./assets/perintah.png)



Setelah repositori berhasil dibuat, GitHub akan menampilkan beberapa instruksi berupa perintah Git yang perlu dijalankan. Perintah-perintah tersebut digunakan untuk menghubungkan proyek yang ada di komputer lokal dengan repositori yang telah dibuat di GitHub.



\### Langkah 3: Membuat Folder Proyek dan File



!\[Langkah 3]
(./assets/file.png)



Pada tahap ini dibuat sebuah folder proyek pada komputer lokal. Di dalam folder tersebut kemudian ditambahkan beberapa file yang nantinya akan diunggah dan disimpan ke dalam repositori GitHub.



\### Langkah 4: Membuka CMD dari Direktori Folder Proyek



!\[Langkah 4]
(assets/cmd.png)



Langkah selanjutnya adalah membuka Command Prompt (CMD) atau terminal dan memastikan bahwa direktori yang aktif berada pada folder proyek yang telah dibuat sebelumnya. Hal ini penting agar setiap perintah Git yang dijalankan dapat diterapkan pada folder proyek yang benar.



\### Langkah 5: Repositori Berhasil Diperbarui



!\[Langkah 6]
(./assets/cek.png)



Langkah terakhir adalah memeriksa repositori pada GitHub untuk memastikan bahwa file yang telah di-\*push\* dari komputer lokal sudah berhasil terunggah dan muncul pada repositori tersebut.
